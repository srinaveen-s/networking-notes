# TCP/IP Model

> Status: ✅ Completed

---

## 📖 Overview

The TCP/IP (Transmission Control Protocol/Internet Protocol) model is the communication model used on the Internet. It explains how devices communicate with each other through a network. Unlike the OSI model, which is mainly used to understand networking concepts, TCP/IP is the model that is actually implemented in real-world networks.

---

## 🎯 Why is TCP/IP Important?

The TCP/IP model allows different computers and devices to communicate using a common set of rules. Every time we browse a website, send an email, watch YouTube, or use WhatsApp, TCP/IP is working in the background.

---

## 🏗 The Four Layers

### Application Layer

This is the top layer of the TCP/IP model. It provides network services directly to applications used by the user.

Common protocols include:
- HTTP
- HTTPS
- FTP
- SMTP
- DNS

---

### Transport Layer

The Transport Layer is responsible for end-to-end communication between two devices.

Its responsibilities include:
- Reliable data delivery
- Error checking
- Flow control
- Data segmentation

Protocols used:
- TCP
- UDP

---

### Internet Layer

The Internet Layer is responsible for logical addressing and routing.

Its responsibilities include:
- Assigning IP addresses
- Finding the best route
- Delivering packets to the destination

Protocols used:
- IP
- ICMP
- ARP

---

### Network Access Layer

This layer is responsible for transmitting data through the physical network.

Examples include:
- Ethernet
- Wi-Fi

---

## 🔄 Difference Between OSI and TCP/IP

The OSI model contains seven layers and is mainly used for learning networking concepts.

The TCP/IP model contains four layers and is used in real computer networks and on the Internet.

---

## 🌍 Real-Life Example

When I open YouTube on my laptop:

- The browser sends an HTTPS request.
- TCP creates a reliable connection.
- IP finds the destination server.
- Ethernet or Wi-Fi transfers the data.
- The server sends the response back.

This entire communication follows the TCP/IP model.
